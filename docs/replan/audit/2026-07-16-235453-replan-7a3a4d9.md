## Per-agent findings

### Codebase alignment
**Critical**
- Init order makes "chooser runs at first `handleResize`, before any asset bundle loads" impossible: the default bundle loads and parses inside `Game.init()` (Game.ts:88), while `handleResize` only runs from `addRef` (Game.ts:360), gated on `#state === 'running'` set at the end of `init()`; the stamp would read an unset fail-loud `pixelScale` and throw at boot.
- `meta.scale: "1"` defeats the resolution stamp in pixi 8.16.0 (`parseFloat("1")` is truthy and overwrites `texture.source.resolution` to 1); `Tileset.from` hardcodes `meta: {scale: '1'}` (Tileset.ts:65-71). Must be omitted, not kept.

**Important**
- `gameScreen.ts` is an unlisted consumer of `BANNER_SLICE`/`FOCUS_RING`/`nineSlice`; `mainMenuScreen` has 5 more `nineSlice` call sites.
- Tiled numeric scaling lives in engine code (`Tileset.from`/`Tilemap.from`/`Map`), not "the pixi-tools parse step"; `Tilemap` fields are readonly; where the multiply happens needed a concrete decision.
- `UiRoot.test.ts`/`Modal.test.ts` ".textures" prediction imprecise: the focus-ring fixture swaps `assetName` for a `texture`.
- The archived spark generator uses Node built-ins, `SIZE = 16`, writes only spark.png, no JSON; needs a rewrite, not a rerun.
- `scale-fnt.mjs` already supports fractional factors (0.25 divides exactly); a "divide mode" is unnecessary.

**Minor**
- World-quantities table, widget constants, bundle aliases (banner-hover/banner-active genuinely unused), baked ×4 uniformity, pixi 8.16.0 version and `fast-png` absence all confirmed exactly.

### Pixi.js fact-check
**Critical**
- `meta.scale: "1"` actively defeats the stamp (Spritesheet.mjs:24-30); must omit the key entirely.
- The stamp must be live before the `Spritesheet` constructor (resolution snapshotted there, not at `.parse()`); the built-in `spritesheetAsset.parse` has no seam between page load and construction, so injection must go through page-texture load options or a replaced parser.
- Setting `source.resolution` after a standalone `Texture` is created does not update `Texture.width` (no resize event emitted); requires construction-time resolution or an explicit `update()`.

**Important**
- The `TextureSource.defaultOptions` rejection rationale is imprecise: `TexturePool` filter textures force resolution 1; the real exposure is `RenderTexture.create` without an explicit resolution.
- Keep the `@Nx` filename ban as an authoring guardrail (Resolver.RETINA_PREFIX would silently set resolution).

**Minor**
- Confirmed: frame rects divided by resolution, `defaultBorders` passed raw; NineSlice math exact with borders × scale; bitmap-text `fontSize / baseMeasurementFontSize` uniform scaling; `.fnt` pages inherit global NEAREST (`loadBitmapFont.mjs:46-51` + Game.ts:73); `fast-png` real and maintained (image-js, v8.0.0).

### Feasibility & risks
**Critical**
- Init-order guarantee inverted (same as alignment); fix by choosing `pixelScale` inside `init()` from viewport device px before `loadBundle`, which also removes the hidden/0-sized-container cliff that would lock scale 2 for a session.
- `meta.scale: "1"` defeats the stamp (same as others), including `Tileset.from`'s hardcoded value.

**Important**
- Existing tests break and were missing from the plan: `playerSystem.test.ts` (tap offsets 78/10, MAX_SPEED velocity asserts), `Tileset.test.ts` (tileWidth 16, boundingBox).
- Tiled scaling layering conflict: stamp must land before `Tileset.from`'s internal `spritesheet.parse()`; readonly `Tilemap` fields block post-processing; needs explicit design.
- Font-page exclusion had no stated mechanism; discriminate via the bundle manifest (font pages are implicit sub-loads).
- `scale-fnt.mjs` divide is silently lossy (`Math.round`); add a divisibility guard that aborts.
- `map.json` `width`/`height` are tile counts; a blind ÷4 corrupts map dimensions. Field-aware divide required.
- UI layout constants (padding/gap/minWidth) were omitted from the scaling story; sweep acceptance would show proportion drift.
- `FOCUS_RING` const must become lazy (resolved texture unavailable at import time); `nineSlice()` signature ripples to 6 call sites across two screens.

**Minor**
- `public/` is git-tracked (git is the real rollback); an in-repo backup dir would bloat git and the Docker image.
- SSR poses no hidden boot path; no existing coupling to raw texture sizes; ~11 fontSize sites; acceptance is necessarily a manual visual check.

### Project standards
**Important**
- The parser stamp does not share the existing parsers' registration shape: `TiledTilemapAsset`/`TiledTilesetAsset` are stateless module consts registered globally at Game.ts module top; "closes over the owning Game at registration" is a new shape that must be stated plainly, registered from within `Game`.
- `MAX_SPEED` migration site incomplete: it is an exported const consumed in `motionSystem` and `playerSystem` (`normalize(MAX_SPEED)`); both use sites need the multiply.
- Parser test fixtures should be inline builders in the existing `Tilemap.test.ts` style; the repo has no fixtures directory.

**Minor**
- `choosePixelScale?` mirrors `focusKeys?` correctly (named type suggested); fail-loud getter has strong engine precedent; `FocusRingOptions` texture injection improves convention conformance; the game-singleton import pattern is established but the touched systems don't import it yet; doc format complies with final-state/Open-decisions conventions.

### Fresh perspective
**Critical**
- The chooser chicken-and-egg (needs container size available only at `addRef`; stamp needs `pixelScale` before `init()`'s parse) is unresolved as written; thread size into `init()` and note the tradeoff with init/asset parallelism.
- The tileset scaling story self-contradicts: `meta.scale: '1'` in `Tileset.from` nullifies the stamp, numeric scaling lives in engine not pixi-tools, and no `pixelScale` source is stated for the unchanged `.from()` signatures; risk of double-scaling or gaps.

**Important**
- Specify how the stamp reaches built-in-parsed assets (no LoaderParser seam; resolution captured at construction); add to spike.
- UI layout-spacing constants omitted; at scale 4 the acceptance check cannot catch a wrong decision (×4 = unchanged), only the sweep can.
- `$/public_1x` contains a partial prior 1× migration: the `monogram` font un-bake is finished and reusable; tileset/character there are stale.
- The FocusRing engine change is bigger than described: `FocusRingOptions` must also drop the four inset fields; verify the atlas focus-ring frame carries borders.

**Minor**
- Section 2 pixi internals accurate; both `.fnt` files divide cleanly by 4; snap threshold appears twice in `motionSystem`; keeping tap input in raw device px is correct.

## Changes applied to plan
- Chooser moved from `handleResize` to the top of `Game.init()`, before `pixi.Assets.init` and the default-bundle load, taking viewport device px (`window.innerWidth/innerHeight × devicePixelRatio`) instead of canvas buffer size; rationale for why `handleResize` is too late stated inline. Named `ChoosePixelScale` type added.
- Stamp mechanism redesigned: construction-time texture load options (`data.resolution`) injected by `Game.init()` into the manifest's art entries, replacing the post-load parser hook; this encodes the Spritesheet-constructor snapshot and standalone `Texture.width` constraints and doubles as the font-page/render-texture exclusion mechanism. `defaultOptions` rejection rationale corrected to `RenderTexture.create`.
- `meta.scale` guidance inverted: the key must be absent everywhere (was "keep meta.scale: '1'"); `Tileset.from` drops its hardcoded `meta: {scale: '1'}`; the generator never emits it.
- Tiled scaling design made concrete: `Tileset.from`/`Tilemap.from` gain an optional `{pixelScale = 1}` options argument passed by the parsers; parsers move registration into `Game.init()` (stated plainly as a new shape, not mirroring the stateless consts); raw-JSON frame extraction kept with an explicit double-scaling warning.
- World-quantities table: `MAX_SPEED` row now names both use sites (`motionSystem`, `playerSystem` `normalize`); snap threshold marked as two occurrences; grep audit widened to UI screens.
- UI section: `@pixi/layout` spacing constants (padding 32/8, gap 16/12, minWidth 220) added to the art-px conversion; consumer list extended with `gameScreen` and the 6 `nineSlice` call sites; `FOCUS_RING` const deleted outright (lazy per-screen construction); `FocusRingOptions` drops the four inset fields; focus-ring atlas frame carries borders; test predictions corrected (`UiRoot.test.ts` swaps assetName for texture) and `playerSystem.test.ts`/`Tileset.test.ts` updates enumerated.
- Migration: backup moved outside the repo (git noted as primary rollback, Docker context noted); PNG audit covers all channels including alpha; numeric ÷4 gets a divisibility guard and is field-aware (`map.json` width/height are tile counts); `scale-fnt.mjs` used at factor 0.25 (divide mode dropped); the finished `$/public_1x` monogram un-bake reused; spark generator explicitly rewritten on fast-png to emit PNG + JSON.
- Testing: spike refocused on manifest-injection reach and standalone `Texture.width` (font-page NEAREST now source-verified); parser tests use inline fixture builders with explicit `{pixelScale}`; acceptance stated as a manual visual check.
- Open decisions: added one item — how system unit tests obtain `game.pixelScale` given the fail-loud singleton getter.
